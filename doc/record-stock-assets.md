# 如何记录股票资产

PanassetLite 支持两种与股票相关的记法：**逐只记股票**（适合单标的、有明确股数）和 **记账户总资产**（适合雪球/券商账户里多只持仓、只关心总额）。选对类别，市值、盈亏和年化才会算对。

## 1. 先选记法

| 你的目标 | 推荐类别 | 流水类型 |
|----------|----------|----------|
| 跟踪 **某一只股票** 的股数、成本、盈亏 | **股票** | 买入、卖出、估值更新 |
| 跟踪 **整个证券账户** 的总资产（多只股合计） | **理财产品** 或 **其他**（也可 **现金存款**） | 存入、取出、估值更新 |
| 账户里还有分红留在户内 | 同上 | 可加 **收益入账** |

**不要用「股票」类型去记「整个账户的总市值」。** 股票类按 **份额 × 单价** 计价；没有买入记录时，即使做了估值更新，市值也会是 0，**近期年化** 也不会显示。

## 2. 逐只记股票（股票类）

### 2.1 新建资产

在 **资产** 页 → **添加资产**：

| 字段 | 说明 |
|------|------|
| 名称 | 如「贵州茅台」 |
| 类别 | **股票** |
| 计价货币 | CNY / USD / HKD 等 |
| 平台 | 券商或渠道，如「华泰证券」「雪球」（可选） |
| 行情来源 | **手动更新估值**，或 Finnhub 自动（美股 ticker，需在设置中配置 key） |

创建资产时 **没有「持有」字段**。「持有」列是系统根据 **买入 / 卖出** 自动算出的股数（或份额）。

### 2.2 流水怎么记

| 流水类型 | 何时使用 | 填什么 |
|----------|----------|--------|
| **买入** | 建仓、加仓 | 数量（股数）、单价 |
| **卖出** | 减仓、清仓 | 数量、单价 |
| **估值更新** | 不想依赖行情、或手动校正总市值 | **当日总市值**（该只股票的市值，不是整户） |

示例：100 股 × 1450 元买入；之后某次估值填 158,500 元（100 股的总市值）。

### 2.3 列表里各列含义

| 列 | 含义 |
|----|------|
| **持有** | 当前股数（买入 − 卖出）；无买入时为「—」 |
| **市值** | 持有 × 单价，或由最近一次估值/行情推导 |
| **累计盈亏** | 相对净投入的本金盈亏 |
| **年化 (XIRR)** | 从历次买卖现金流到当前市值的内部收益率 |
| **近期年化** | 股票类 **不显示**（—） |

## 3. 记整个账户总资产（理财 / 其他 / 现金存款）

典型场景：**雪球账户里买了多只股票，只想记录账户里的全部股票合计多少钱。**

### 3.1 新建资产

| 字段 | 示例 |
|------|------|
| 名称 | 雪球股票账户 |
| 类别 | **理财产品** 或 **其他**（**现金存款** 亦可） |
| 平台 | 雪球 |
| 行情来源 | **手动更新估值** |

### 3.2 流水怎么记

| 流水类型 | 何时使用 | 填什么 |
|----------|----------|--------|
| **存入** | 首次录入、或从银行卡转入账户的本金 | 金额 |
| **取出** | 从账户转出到银行卡 | 金额 |
| **估值更新** | 定期同步账户总资产 | 当日在雪球看到的 **总市值** |
| **收益入账** | 可选：分红到账且仍留在账户内 | 金额 |

**「存入」和「估值更新」不要混为一谈：** 存入表示你新投入的本金；估值更新表示某时刻账户里已有的总价值（含浮盈浮亏）。

### 3.3 列表示例

| 日期 | 类型 | 明细 | 说明 |
|------|------|------|------|
| 2026-02-10 | 存入 | 619,773 CNY | 首次录入时的账户总资产（或累计净投入） |
| 2026-05-17 | 估值更新 | 622,310 CNY | 同步雪球总市值 |
| 2026-06-12 | 估值更新 | 623,668 CNY | 再次同步 |

此例中：

- **累计盈亏** ≈ 623,668 − 619,773 = **+3,895**
- **近期年化**（最近两次估值之间）≈ (623,668 − 622,310) ÷ 622,310 × (365 ÷ 26) ≈ **+3.1%**
- **年化 (XIRR)** 会从首次存入算到当前市值，跨度更长，数值通常与近期年化不同

### 3.4 列表里各列含义

| 列 | 含义 |
|----|------|
| **持有** | 总值类资产固定为「—」（无股数概念） |
| **市值** | 最近一次 **估值更新** 的总值 |
| **累计盈亏** | 市值 + 累计取出 − 累计存入 |
| **年化 (XIRR)** | 含存入/取出与当前市值的现金流年化 |
| **近期年化** | 最近 **两次估值更新** 之间的区间年化；区间内若有存入/取出会自动扣除 |

近期年化不显示时，请检查：① 类别是否为股票/基金/加密/负债；② 是否至少有两笔 **估值更新**（存入不算）；③ 两次估值是否同一天。

## 4. 常见问题

### 「持有」该填什么？

**不用填。** 它是系统算出来的：股票类 = 买入卖出后的股数；账户总额类 = 无意义，显示「—」即可。

### 我已经建了「股票」类，只想记总市值怎么办？

建议 **编辑资产**，把类别改为 **理财产品** 或 **其他**，流水保持「存入 + 估值更新」即可。

若必须保留「股票」类别，可用权宜之计：记一笔 **买入**，数量填 `1`、单价填总本金，之后用 **估值更新** 填账户总市值。此时「持有」会显示 1，没有实际含义，**近期年化** 仍不会显示。

### 港股、美股怎么记？

- **逐只记**：建对应币种（HKD / USD）的股票资产；美股可配 Finnhub 自动行情。
- **只记账户总额**：币种选账户主币种，用 **估值更新** 填该币种下的总市值；多币种账户可拆成多个资产，或在设置中确认汇率折算是否符合你的习惯。

### 和演示数据的区别

设置 → **加载演示数据** 中的「贵州茅台」「宁德时代」等是 **逐只股票** 示例（买入 + 估值更新）。若你的需求是 **整户总资产**，请参考本文第 3 节，不要照搬演示里单只股票的记法。
