# 如何记录加密货币资产

PanassetLite 把加密货币当作 **份额 × 单价** 计价（与股票、基金相同）。你可以 **逐币种跟踪持仓**，也可以只记 **交易所账户总资产**。选对类别，市值、盈亏和年化才会算对。

## 1. 先选记法

| 你的目标 | 推荐类别 | 流水类型 |
|----------|----------|----------|
| 跟踪 **某一种币** 的持仓数量、成本、盈亏 | **加密货币** | 买入、卖出、估值更新 |
| 跟踪 **整个交易所账户** 的总资产（多币种合计） | **理财产品** 或 **其他** | 存入、取出、估值更新 |
| 单独跟踪 **定投计划**（与现货账户分开） | **策略**（归属到对应加密资产） | 策略流水：存入、取出、估值更新 |

**不要用「加密货币」类型去记「整个账户的总市值」。** 加密类按 **持有数量 × 单价** 计价；没有买入记录时，即使做了估值更新，市值也会是 0。

## 2. 逐币种记持仓（加密货币类）

典型场景：在币安持有 0.08 BTC、2000 USDT，想分别看清每种币的盈亏。

### 2.1 新建资产

在 **资产** 页 → **添加资产**：

| 字段 | 说明 |
|------|------|
| 名称 | 如「比特币」「USDT」 |
| 类别 | **加密货币** |
| 计价货币 | 建议 **CNY**（见下文「自动行情」说明） |
| 平台 | 交易所或钱包，如「币安」「OKX」（可选） |
| 行情来源 | **CoinGecko 自动（免 key）**，或 **手动更新估值** |
| 行情代码 | CoinGecko id，如 `bitcoin`、`ethereum`、`tether`（不是交易所 ticker） |

创建资产时 **没有「持有」字段**。「持有」列由 **买入 / 卖出** 自动算出当前数量。

#### 如何查 CoinGecko id

在 [coingecko.com](https://www.coingecko.com/) 搜索币种，打开详情页，地址栏中的 slug 即为 id，例如：

- 比特币 → `bitcoin`
- 以太坊 → `ethereum`
- USDT → `tether`
- Solana → `solana`

### 2.2 流水怎么记

| 流水类型 | 何时使用 | 填什么 |
|----------|----------|--------|
| **买入** | 建仓、加仓、收到转账 | **数量**（币数）、**单价**（该币当时的人民币单价） |
| **卖出** | 减仓、清仓、转出 | 数量、单价 |
| **估值更新** | 不用自动行情、或手动校正总市值 | **当日该币种的总市值**（数量 × 单价，不是整户） |

示例（与演示数据类似）：

| 日期 | 类型 | 明细 | 说明 |
|------|------|------|------|
| 2025-10-01 | 买入 | 0.05 BTC @ ¥430,000 | 首次建仓 |
| 2026-03-09 | 买入 | 0.03 BTC @ ¥480,000 | 加仓 |
| 2026-04-06 | 买入 | 2000 USDT @ ¥7.25 | 稳定币单独建资产 |

买入时的 **单价** 填你实际成交的人民币均价（或按当时汇率折算）。系统用「数量 × 单价」累计净投入，并据此算盈亏与 XIRR。

### 2.3 自动行情（CoinGecko）

在 **设置 → 行情 API** 点击 **自动获取加密货币行情**，会为所有行情来源选 CoinGecko 的资产拉取 **人民币单价**，按日保存在本地（同一天重复获取以最后一次为准）。

要点：

- **无需 API key**，联网即可用。
- 接口返回的是 **CNY 单价**；使用自动行情时，计价货币建议选 **CNY**，买入/卖出单价也按人民币填写。
- 若某币种 id 填错或 CoinGecko 未收录，该资产会拉取失败，可改用手动 **估值更新** 或核对 id。

不点自动获取时，市值沿用最近一次买入价、估值更新或已保存的行情；建议定期刷新或手动估值。

### 2.4 列表里各列含义

| 列 | 含义 |
|----|------|
| **持有** | 当前币数（买入 − 卖出） |
| **市值** | 持有 × 单价（来自行情或估值/成交价推导） |
| **累计盈亏** | 相对净投入的本金盈亏 |
| **年化 (XIRR)** | 从历次买卖现金流到当前市值的内部收益率 |
| **近期年化** | 加密类 **不显示**（—），与股票、基金相同 |

## 3. 记整个交易所账户总资产

典型场景：币安里持有 BTC、ETH、USDT 等多种币，只关心 **账户总权益**，不想逐币种维护。

### 3.1 新建资产

| 字段 | 示例 |
|------|------|
| 名称 | 币安现货账户 |
| 类别 | **理财产品** 或 **其他** |
| 平台 | 币安 |
| 行情来源 | **手动更新估值** |

### 3.2 流水怎么记

| 流水类型 | 何时使用 | 填什么 |
|----------|----------|--------|
| **存入** | 首次录入、或从银行卡/其他账户划入的本金 | 金额 |
| **取出** | 从交易所提现到银行或其他账户 | 金额 |
| **估值更新** | 定期同步账户总权益 | 当日 App 里看到的 **总资产（约人民币）** |

**「存入」和「估值更新」不要混为一谈：** 存入表示新投入的本金；估值更新表示某时刻账户里已有的总价值（含浮盈浮亏）。

此记法下 **近期年化** 可用（至少两笔 **估值更新**），适合判断这段时间账户涨跌幅。

## 4. 策略流水（可选）

若你有 **定投** 等习惯，希望与「现货账户里实际持有多少」分开记账，可使用 **策略** 功能：

1. 先建好对应的 **加密货币** 资产（如「比特币」）。
2. 在 **策略** 页新建策略，**归属资产** 选该币种，类型如「定投」。
3. 在策略下记 **存入 / 取出 / 估值更新**，不影响资产侧的买入卖出。

演示数据中的「BTC 周定投」即此用法：资产侧记现货买入，策略侧单独跟踪定投计划的投入与表现。

## 5. 常见问题

### 买入单价填总价还是单价？

填 **单价**（每 1 个币的人民币价格）。数量 × 单价 = 该笔成本。若只记得一笔总花费，用「总花费 ÷ 币数」得到单价。

### 链上钱包、冷钱包怎么记？

与交易所相同：为每种要跟踪的币建 **加密货币** 资产，用 **买入** 记录当前持有（数量 + 成本单价），之后用 CoinGecko 自动行情或 **估值更新** 维护市值。平台字段可填「Ledger」「自托管」等。

### 稳定币 USDT / USDC 要单独建资产吗？

建议 **每种稳定币单独一条资产**（如 USDT 用 id `tether`），与 BTC、ETH 分开记买入与市值。若只关心法币本位总权益，用第 3 节「整户总资产」更简单。

### 我已经建了「加密货币」类，只想记账户总额怎么办？

**编辑资产**，把类别改为 **理财产品** 或 **其他**，流水改为「存入 + 估值更新」即可。

### 和演示数据的区别

设置 → **加载演示数据** 中的「比特币」「USDT」是 **逐币种 + CoinGecko 自动行情** 示例；「BTC 周定投」是 **策略** 示例。若你只关心交易所总权益，请参考本文第 3 节，不要照搬逐币种记法。
